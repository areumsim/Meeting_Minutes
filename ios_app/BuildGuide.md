# AI 회의록 모바일 앱 — 빌드부터 iPhone 설치까지 완전 가이드

> **아키텍처 변경 안내**: 이 앱은 이제 완전히 독립된 **서버리스(Serverless)** 앱으로 작동합니다. 기존 버전과 달리 PC 등에 백엔드 서버(파이썬)를 띄워둘 필요가 **전혀 없습니다**. 오직 스마트폰과 인터넷 연결, 그리고 OpenAI API Key만 있으면 내 폰 자체가 독자적인 AI 회의록 시스템이 됩니다.

## 1. 전체 흐름 한눈에 보기

```text
Windows PC                          Mac                              iPhone (독립 동작)
──────────                          ───                              ──────

① 앱 UI/UX 코드 작성
② npm run build      ─(이동)─▶  ③ 코드를 Mac으로 복사
                                ④ Mac에서 의존성 셋업
                                ⑤ npx cap sync ios
                                ⑥ Xcode 열고 기기선택
                                ⑦ ▶ Run 클릭 ─────────(USB)──────▶ ⑧ 앱 강제 설치!
                                                                    ⑨ "신뢰" 설정
                                                                    ⑩ 실행 후 API Key 직접 등록
                                                                     (설치 끝! 이후 PC나 Mac 불필요)
```

**소요 시간**: 환경 구축 약 1시간. (이후 수정사항 재배포는 1분 컷)

## 3. Windows PC에서 선행 작업

개발이 완료된 웹 프론트엔드 위치에서 프로덕션 빌드를 수행합니다.

```powershell
cd C:\프로젝트경로\web\frontend

# 의존성 모듈 설치
npm install

# 프로덕션 빌드 (dist/ 폴더 생성점)
npm run build
```

이 작업이 완료되면 안의 `dist/` 폴더 내부에 HTML/CSS/JS 가 전부 압축되어 들어갑니다. 바로 이 `dist` 폴더가 스마트폰 안에 심어져 동작하는 핵심 두뇌가 됩니다.

## 4. 코드를 Mac으로 옮기기

가장 쉬운 방법은 GitHub를 쓰는 것입니다. 하지만 어려우면 USB나 클라우드(Google Drive, Send Anywhere 등)를 통해 **`web/frontend` 디렉토리 통째로 Mac으로 복사**합니다.

## 5. Mac 초기 설정 (최초 1회)

Mac을 처음 셋업한다고 가정합니다.

1. **Xcode 설치**: App Store에서 12GB짜리 Xcode를 받아 **반드시 한 번 실행**하여 약관을 동의합니다.
2. **명령줄 도구**: 터미널을 열고 `xcode-select --install` 엔터.
3. **Node.js 설치**: 맥용 Node 20.x 버전을 설치합니다.
4. **CocoaPods 설치**: 터미널에 `sudo gem install cocoapods` 입력.

## 6. Mac에서 iOS 빌드 준비

Mac으로 옮겨온 프로젝트 폴더 터미널에서 아래 명령어들을 차례대로 입력합니다.

```bash
cd 프로젝트경로/web/frontend

# 모듈이 꼬였을 수 있으니 다시 설치
npm install

# Mac 버전으로 iOS 폴더 뼈대 심기
npx cap add ios

# 방금 전 Windows에서 빌드한 코드를 iOS 껍데기에 밀어넣기
npx cap sync ios
```

이 명령어들이 문제없이 성공하면 `ios/` 라는 폴더가 떡하니 생기며 그 안에 각종 Xcode용 파일이 준비됩니다.

## 7. Xcode에서 iPhone에 설치하기

### 7-1. Xcode로 열기
터미널에서 명령어 입력:
```bash
npx cap open ios
```
(혹은 Finder에서 새로 생긴 `ios/App/App.xcworkspace` 더블클릭)

### 7-2. 개발자 로그인 및 서명(Signing)
1. Xcode 상단 메뉴 `Xcode` -> `Settings` -> `Accounts` 에서 본인의 일반 모바일용 Apple ID로 로그인을 마칩니다.
2. 왼쪽 파일 목록의 파란 아이콘 `App` 최상단을 클릭합니다.
3. 가운데 메인 화면에서 탭 메뉴 중 `Signing & Capabilities`를 클릭합니다.
4. `Team` 항목의 드롭다운을 열어 방금 로그인한 본인 이름 `(Personal Team)`을 선택합니다.
5. `Bundle Identifier`를 유일하게 조금 바꿉니다. (예: `com.나만의이름.minutes`)

### 7-3. 마이크 권한 설정 (필수 🔥)
iOS 정책상 마이크를 쓰려면 권한 명시가 필수입니다. 추가하지 않으면 녹음 버튼을 누르는 순간 앱이 바로 튕깁니다(강제 종료).
1. Xcode의 제일 왼쪽 파일 트리를 엽니다.
2. `App` ➔ `App` ➔ **`Info.plist`** 파일을 클릭합니다.
3. 파일 안쪽 빈 공간에 마우스 우클릭 -> `Add Row` 클릭.
4. 새로 생긴 줄의 왼쪽 Key 에 **`Privacy - Microphone Usage Description`** 을 정확히 선택(타이핑)합니다.
5. 중간 Type은 `String`으로 두고, 맨 오른쪽 Value에 **`실시간 회의록 음성 인식을 위해 마이크 접근 권한이 필요합니다.`** 라고 한글로 적습니다.

### 7-3b. 백그라운드 오디오 설정 (권장)

녹음 중 화면이 꺼지거나 다른 앱으로 잠시 전환해도 녹음이 끊기지 않도록 설정합니다.

1. Xcode 왼쪽 파일 트리에서 파란 아이콘 **`App`** 최상단 클릭.
2. 가운데 메인 화면에서 **`Signing & Capabilities`** 탭 선택.
3. 좌측 상단 **`+ Capability`** 버튼 클릭.
4. 검색창에 **`Background Modes`** 입력 후 더블 클릭하여 추가.
5. 추가된 Background Modes 섹션에서 **`Audio, AirPlay, and Picture in Picture`** 항목에 체크.

이 설정 없이도 앱은 동작하지만, **녹음 중 화면 잠금이나 앱 전환 시 녹음이 중단**됩니다.

### 7-4. USB 꽂고 빌드 슛(Run)!
1. iPhone을 Mac과 USB 케이블로 연결합니다.
2. 아이폰 화면에 "이 컴퓨터를 신뢰하겠습니까?" 창이 뜨면 즉시 `신뢰`를 누릅니다.
3. Xcode 정중앙 맨 위의 **[App > 시뮬레이터 목록]** 을 눌러 제일 위에 뜬 진짜 기기(예: `나의 iPhone`)로 목적지를 바꿉니다.
4. Xcode 맨 왼쪽 위에 있는 **▶ (재생 버튼 모양, Run)** 을 누릅니다.

상단에 `Building...` 게이지가 차오르다가 `Build Succeeded`가 뜨면, 내 호주머니 속 아이폰 화면에 앱이 툭! 하고 강제로 깔리면서 켜집니다.

## 8. iPhone에서 앱 신뢰 설정 (매우 중요)

무료 Apple ID로 앱을 처음 심었기 때문에 아이폰 보안 시스템이 앱 실행을 잠시 막습니다. 앱 아이콘을 눌렀을 때 `신뢰하지 않는 개발자`라는 에러창이 뜨면 당황하지 마세요.

1. 아이폰의 **일반 [설정] 앱** 접속
2. **[일반]** -> **[VPN 및 기기 관리]**
3. 하단 "개발자 앱" 목록에 내 Apple ID 이메일 클릭
4. **`[나의 이메일] 신뢰` 파란 글씨 터치** -> 팝업에서 `신뢰` 또 한 번 터치.

그 다음부터는 바탕화면에서 앱을 탭하면 정상적으로 켜집니다!
이제 앱 내부의 Settings 로 들어가 **OpenAI API Key**를 입력한 후 사용하면, PC 없이 카페나 이동 중에도 아이폰의 마이크로 이 앱을 완전 독립적으로 구동할 수 있습니다.

## 9. 자주 묻는 질문 / 트러블슈팅

### 왜 PC 서버 설정하는 란이 없어졌나요?
기존에 존재하던 "서버 URL 접속 테스트" 기능은 이제 앱이 단독으로 연산하게 아키텍처를 뒤집어 엎었으므로 모두 폐기되었습니다. 백엔드(FastAPI)가 없으니 연결 문제로 신경 쓰실 일이 원천 차단되었습니다.

### 테스트 기간 제한?
무료 Apple ID 빌드는 단점이 "앱 인증 수명이 7일" 이라는 점입니다. 7일이 지나면 앱을 눌러도 강제로 튕깁니다. 그때는 폰을 맥에 다시 연결하시고 Xcode에서 "▶ Run"을 한 번 더 눌러 덮어쓰기 재설치를 해주시면 다시 7일간 부활합니다. 이게 번거로우실 땐 $99/년 을 내고 애플 개발자 프로그램으로 업그레이드 하시면 1년 단위로 연장됩니다.

### CocoaPods 에러 (Command PhaseScriptExecution failed) 나올 때
Mac M1, M2 에서 자주 발생합니다. 터미널을 열고 iOS 폴더로 들어가서 Pods를 완전히 리셋해주세요.
```bash
cd ios/App
pod deintegrate
pod install
cd ../..
npx cap sync ios
```
이후 Xcode 가셔서 다시 Run 하면 정상화됩니다.
